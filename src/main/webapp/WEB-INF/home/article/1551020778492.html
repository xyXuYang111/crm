log4j2日志的完整配置文件信息和同步与异步的配置信息
2019-02-24 23:06 我的电脑 1644

log4j2日志的完整配置文件信息和同步与异步的配置信息，如果在springboot中用，请先去掉默认的logback日志

文件名:log4j2.xml

  1. <?xml version="1.0" encoding="UTF-8"?>
  2. <!--  status="OFF"，可以去掉，它的含义为是否记录log4j2本身的event信息，默认是OFF -->
  3. <configuration status="OFF">
  4.     <!-- 定义下面的引用名 -->
  5.     <!-- 
  6.         自定义格式：
  7.             %t：线程名称
  8.             %p：日志级别
  9.             %c：日志消息所在类名
  10.             %m：消息内容
  11.             %M：输出执行方法
  12.             %d：发生时间，%d{yyyy-MM-dd HH:mm:ss,SSS}，输出类似：2018-10-18 22:10:28,921
  13.             %x: 输出和当前线程相关联的NDC(嵌套诊断环境),尤其用到像java servlets这样的多客户多线程的应用中。
  14.             %L：代码中的行数
  15.             %n：换行 
  16.         --> 
  17.     <Properties>
  18.         <property name="log_pattern">%d{yyyy-MM-dd HH:mm:ss z} %-5p %c{36} %L %M - %m%x%n</property>
  19.         <property name="file_name">logs/app.log</property>
  20.         <property name="file_info">logs/info.log</property>
  21.         <property name="rolling_file_name">logs/app-%d{yyyy-MM-dd}-%i.log.gz</property>
  22.         <property name="every_file_size">10M</property><!-- 日志切割的最小单位 -->
  23.         <property name="output_log_level">debug</property><!-- 日志输出级别 -->
  24.     </Properties>
  26.     <!--先定义所有的appender-->
  27.     <!-- 日志级别
  28.             trace：追踪，就是程序推进一下，可以写个trace输出
  29.             debug：调试，一般作为最低级别，trace基本不用。
  30.             info：输出重要的信息，使用较多
  31.             warn：警告，有些信息不是错误信息，但也要给一些提示。
  32.             error：错误信息。用的也很多。
  33.             fatal：致命错误。级别较高.
  34.      -->
  35.     <appenders>
  37.         <!--Appenders节点，常见的有三种子节点:Console、RollingFile、File.-->
  39.         <!-- Console
  40.         name :给loggers调用
  41.         target:SYSTEM_OUT 或 SYSTEM_ERR,一般只设置默认:SYSTEM_OUT.
  42.         -->
  43.         <Console name="Console" target="SYSTEM_OUT">
  44.             <!--控制台只输出level及以上级别的信息（onMatch），其他的直接拒绝（onMismatch）-->
  45.             <ThresholdFilter level="trace" onMatch="ACCEPT" onMismatch="DENY"></ThresholdFilter>
  46.             <!--输出日志的格式-->
  47.             <PatternLayout pattern="${log_pattern}"></PatternLayout>
  48.         </Console>
  51.          <!--File
  52.          fileName:指定输出日志的目的文件带全路径的文件名.
  53.          append:决定是否添加还是重新建文件会打印出所有信息，false这个log每次运行程序会自动清空，
  54.           -->
  55.             <File name="log" fileName="${file_name}" append="false" >
  56.                 <PatternLayout pattern="${log_pattern}"></PatternLayout>
  57.                  <!--输出level及以上级别的信息（onMatch），其他的直接拒绝（onMismatch）-->
  58.                 <ThresholdFilter level="error" onMatch="ACCEPT" onMismatch="DENY"></ThresholdFilter>
  59.             </File> 
  60.              <File name="logss" fileName="${file_info}" append="false" >
  61.                 <PatternLayout pattern="${log_pattern}"></PatternLayout>
  62.                 <ThresholdFilter level="info" onMatch="ACCEPT" onMismatch="DENY"></ThresholdFilter>
  63.             </File>
  66.         <!--RollingFile
  67.         　              fileName:指定输出日志的目的文件带全路径的文件名.
  68. 　　　　　　　　PatternLayout:输出格式，不设置默认为:%m%n.
  69. 　　　　　　　　filePattern:指定新建日志文件的名称格式.
  70. 　　　　　　　　Policies:指定滚动日志的策略，就是什么时候进行新建日志文件输出日志.
  71. 　　　　　　　   TimeBasedTriggeringPolicy:Policies子节点，基于时间的滚动策略，interval属性用来指定多久滚动一次，默认是1 hour。modulate=true用来调整时间：比如现在是早上3am，interval是4，那么第一次滚动是在4am，接着是8am，12am...而不是7am.
  72. 　　　　　　　　SizeBasedTriggeringPolicy:Policies子节点，基于指定文件大小的滚动策略，size属性用来定义每个日志文件的大小.
  73. 　　　　　　　　DefaultRolloverStrategy:用来指定同一个文件夹下最多有几个日志文件时开始删除最旧的，创建新的(通过max属性),默认
  75.               这个会打印出所有的信息，每次大小超过size，则这size大小的日志会自动存入按年份-月份建立的文件夹下面并进行压缩，作为存档
  76.            -->
  77.        <!-- 按月生成归档日志，可以使用 filePattern="logs/$${date:yyyy-MM}/app-%d{MM-dd-yyyy}-%i.log.gz" 
  79.         <RollingFile name="RollingFile" fileName="${file_name}"
  80.                      filePattern="${rolling_file_name}">
  81.             <PatternLayout pattern="${log_pattern}"></PatternLayout>
  82.             <SizeBasedTriggeringPolicy size="${every_file_size}"></SizeBasedTriggeringPolicy>
  83.         </RollingFile>
  84.         -->
  85.         <!--如果需要配置多个Rollingfile地址，还需要在root下添加appender-ref ref="RollingFile1"/>  
  86.          <RollingFile name="RollingFile1" fileName="logs/app1.log"
  87.                      filePattern="logs/app1-%d{yyyy-MM-dd}-%i.log.gz">
  88.             <PatternLayout pattern="%d{yyyy-MM-dd HH:mm:ss z} %-5level %class{36} %L %M - %msg%xEx%n"></PatternLayout>
  89.             <SizeBasedTriggeringPolicy size="10MB"></SizeBasedTriggeringPolicy>
  90.         </RollingFile>
  91.          -->
  92.   </appenders>
  93.    <!--定义logger
  94.         只有定义了logger并引入的appender，appender才会生效
  95.              常见的有两种:Root和Logger.
  96. 　　　　　　　Root节点用来指定项目的根日志，如果没有单独指定Logger，那么就会默认使用该Root日志输出
  97. 　　　　　　　　　level:日志输出级别，共有8个级别，按照从低到高为：All < Trace < Debug < Info < Warn < Error < Fatal < OFF.
  98. 　　　　　　　　　AppenderRef：Root的子节点，用来指定该日志输出到哪个Appender.
  99. 　　　　　　　Logger节点用来单独指定日志的形式，比如要为指定包下的class指定不同的日志级别等。
  100. 　　　　　　　　　level:日志输出级别，共有8个级别，按照从低到高为：All < Trace < Debug < Info < Warn < Error < Fatal < OFF.
  101. 　　　　　　　　　name:用来指定该Logger所适用的类或者类所在的包全路径,继承自Root节点.
  102. 　　　　　　　　　AppenderRef：Logger的子节点，用来指定该日志输出到哪个Appender,
  103.                                 如果没有指定，就会默认继承自Root.如果指定了，
  104.                                 那么会在指定的这个Appender和Root的Appender中都会输出，
  105.                                 此时我们可以设置Logger的additivity="false"只在自定义的Appender中进行输出。
  106.     -->
  107.     <loggers>
  108.         <!--建立一个默认的root的logger，需要在root的level中指定输出的级别，-->
  109.           <Root level="trace" includeLocation="true">
  110.         <AppenderRef ref="log"></AppenderRef>
  111.        <!-- <AppenderRef ref="Console"></AppenderRef> --> 
  112.         </Root>
  114.         <!-- 异步日志在程序的classpath需要加载disruptor-3.0.0.jar或者更高的版本。
  116.         异步日志分为两种：
  118.         a.全异步模式
  119.         这种异步日志方式，不需要修改修改原理的配置文件，Logger仍然使用<root> and <logger>
  120.         只需要在主程序代码开头，加一句系统属性的代码：
  121.         System.setProperty("Log4jContextSelector", "org.apache.logging.log4j.core.async.AsyncLoggerContextSelector");
  123.         b.异步和非异步混合输出模式
  124.          在配置文件中Logger使用<asyncRoot> or <asyncLogger>
  125.              -->
  126.  </loggers>
  127. </configuration>
